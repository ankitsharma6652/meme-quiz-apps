### Step 3: Set Environment Variables

After deployment starts, go to your service dashboard:

1. Click on your **"mememaster"** service
2. Go to **"Environment"** tab
3. Add these variables:

| Key | Value | How to Get |
|-----|-------|------------|
| `GOOGLE_CLIENT_ID` | Your Google OAuth Client ID | From Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | Your Google OAuth Client Secret | From Google Cloud Console |

**Note**: `SECRET_KEY` and `DATABASE_URL` are auto-generated by Render!

### Step 4: Update Google OAuth Redirect URI

1. Go to https://console.cloud.google.com/
2. Select your project
3. Go to **"Credentials"**
4. Click on your OAuth 2.0 Client ID
5. Add to **"Authorized redirect URIs"**:
   ```
   https://mememaster.onrender.com/authorize/google
   ```
   (Replace with your actual Render URL)
6. Click **"Save"**

### Step 5: Initialize Database (First Time Only)

After first deployment:

1. In Render dashboard, go to your service
2. Click **"Shell"** tab
3. Run this command:
   ```python
   python -c "from app import app, db; app.app_context().push(); db.create_all(); print('Database initialized!')"
   ```

## ğŸ› Troubleshooting

### Deployment Failed?
1. Check **"Events"** tab for error messages
2. Check **"Logs"** tab for detailed logs
3. Make sure all environment variables are set

### Database Connection Error?
1. Make sure PostgreSQL database is created
2. Check that `DATABASE_URL` is automatically set
3. Run database initialization command

### OAuth Not Working?
1. Verify redirect URI in Google Console
2. Check `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are set
3. Make sure you're using HTTPS (Render provides this automatically)
